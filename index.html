¿Qué es MQTT?

Definición

MQTT (Message Queuing Telemetry Transport) es un protocolo de mensajería ligero diseñado específicamente para dispositivos IoT con recursos limitados y conexiones de red inestables.

Utiliza un modelo de publicación/suscripción (pub/sub) donde los dispositivos publican mensajes en "topics" y otros dispositivos se suscriben a esos topics para recibir información.

Ligero

Mínimo overhead, ideal para IoT

Pub/Sub

Modelo desacoplado y escalable

Confiable

Niveles de QoS garantizados

¿Por qué usar MQTT?

  • Eficiencia: Requiere muy poco ancho de banda y recursos, perfecto para dispositivos con batería limitada.
  • Desacoplamiento: Los dispositivos no necesitan conocerse entre sí, solo el broker MQTT.
  • Escalabilidad: Fácil agregar nuevos dispositivos sin modificar los existentes.
  • Confiabilidad: Niveles de QoS (0, 1, 2) garantizan la entrega de mensajes según la importancia.
  • Simplicidad: Fácil de implementar y entender, ideal para proyectos educativos.

Tutorial: Cómo Usar MQTT

1

Crear Cuenta en HiveMQ Cloud

Para este proyecto necesitas crear una cuenta en HiveMQ Cloud para sincronizar el MQTT. Es gratuito y perfecto para proyectos educativos.

Pasos para crear cuenta en HiveMQ Cloud:

  1. Visita hivemq.com/cloud
  2. Haz clic en "Start Free" o "Sign Up"
  3. Completa el formulario de registro con tu email
  4. Verifica tu email (revisa tu bandeja de entrada)
  5. Inicia sesión en tu cuenta
  6. Crea un nuevo cluster (elige la opción gratuita)
  7. Selecciona la región más cercana a ti
  8. Una vez creado, ve a "Access Management" → "Credentials"
  9. Crea un nuevo usuario con contraseña
  10. Copia la URL del broker (formato: wss://xxxxx.s1.eu.hivemq.cloud:8884/mqtt)
  11. Guarda tu username y password de forma segura

Importante: El plan gratuito permite hasta 25 conexiones simultáneas, lo cual es suficiente para este proyecto.

Alternativas de Broker:

  • HiveMQ Cloud (recomendado para este proyecto)
  • Mosquitto (broker local gratuito)
  • AWS IoT Core
  • Azure IoT Hub
1.5

Crear Cuenta en Firebase

Para visualizar las bases de datos y usar Firebase Realtime Database, necesitas crear una cuenta en Firebase.

Pasos para crear cuenta en Firebase:

  1. Visita console.firebase.google.com
  2. Haz clic en "Comenzar" o "Get Started"
  3. Inicia sesión con tu cuenta de Google (o crea una si no tienes)
  4. Haz clic en "Agregar proyecto" o "Add project"
  5. Ingresa un nombre para tu proyecto (ej: "iot-mqtt-project")
  6. Acepta los términos y condiciones
  7. Opcional: Desactiva Google Analytics si no lo necesitas
  8. Haz clic en "Crear proyecto"
  9. Espera a que se cree el proyecto (puede tardar unos minutos)
  10. En el menú lateral, ve a "Realtime Database"
  11. Haz clic en "Crear base de datos"
  12. Elige la ubicación más cercana
  13. Para desarrollo, inicia en "modo de prueba" (puedes cambiar las reglas después)
  14. Copia la URL de la base de datos (formato: https://tu-proyecto-default-rtdb.firebaseio.com/)

Configurar Reglas de Seguridad:

En "Reglas" de Realtime Database, para desarrollo puedes usar: 

{
  "rules": {
    ".read": true,
    ".write": true
  }
}

Nota: Estas reglas permiten lectura/escritura a todos. Para producción, configura reglas más restrictivas.

Ventajas de Firebase: Base de datos en tiempo real, sincronización automática, fácil integración con aplicaciones web, y plan gratuito generoso.

2

Configurar Credenciales MQTT

Edita el archivo env/credential.js con las credenciales que obtuviste de HiveMQ Cloud.

Datos que necesitas de HiveMQ Cloud:

  • URL del broker (ej: wss://xxxxx.s1.eu.hivemq.cloud:8884/mqtt)
  • Username (el que creaste en Access Management)
  • Password (la contraseña que configuraste)
const mqttConfig = {
  client_hogar: {
    // URL de HiveMQ Cloud (copia desde tu dashboard)
    url: "wss://xxxxx.s1.eu.hivemq.cloud:8884/mqtt",
    // Username que creaste en HiveMQ Cloud
    username: "tu_usuario_hivemq",
    // Password que configuraste
    password: "tu_contraseña_hivemq",
    options: {
      clean: true,
      connectTimeout: 4000,
      reconnectPeriod: 1000
    }
  },
  client_tren: {
    url: "wss://xxxxx.s1.eu.hivemq.cloud:8884/mqtt",
    username: "tu_usuario_hivemq",
    password: "tu_contraseña_hivemq",
    options: {
      clean: true,
      connectTimeout: 4000,
      reconnectPeriod: 1000
    }
  }
};

Seguridad: No compartas tus credenciales públicamente. El archivo env/credential.js debería estar en .gitignore si usas Git.

3

Conectar un Cliente

Usa la función createMqttClient() para crear una conexión. 

// Incluir el script de credenciales
<script src="../env/credential.js"></script>

// Crear el cliente
const client = createMqttClient('client_hogar');

// Escuchar eventos
client.on('connect', () => {
  console.log('✅ Conectado!');
});

client.on('error', (error) => {
  console.error('❌ Error:', error);
});
4

Publicar Mensajes (Publisher)

Para enviar mensajes, usa el método publish(). 

// Publicar un mensaje
client.publish('hogar/living/luz/set', 'on', { qos: 1 }, (err) => {
  if (err) {
    console.error('Error:', err);
  } else {
    console.log('✅ Mensaje publicado');
  }
});

// Sintaxis: client.publish(topic, mensaje, opciones, callback)

Tip: El parámetro qos (Quality of Service) puede ser 0, 1 o 2. QoS 1 garantiza que el mensaje se entregue al menos una vez.

5

Suscribirse a Topics (Subscriber)

Para recibir mensajes, suscríbete a un topic usando subscribe(). 

// Suscribirse a un topic específico
client.subscribe('hogar/living/luz/state', { qos: 1 }, (err, granted) => {
  if (err) {
    console.error('Error:', err);
  } else {
    console.log('✅ Suscrito a:', granted);
  }
});

// Suscribirse a múltiples topics con wildcard
client.subscribe('hogar/#', { qos: 1 }); // # = todos los subtopics
client.subscribe('hogar/+/luz/+', { qos: 1 }); // + = cualquier nivel

Wildcards: Usa # para todos los subtopics y + para un nivel específico.

6

Recibir Mensajes

Escucha el evento message para recibir mensajes. 

client.on('message', (topic, message) => {
  // message es un Buffer, conviértelo a string
  const msg = message.toString();
  
  console.log(`📨 Mensaje recibido: ${topic} = ${msg}`);
  
  // Procesar el mensaje según el topic
  if (topic === 'hogar/living/luz/state') {
    actualizarLuz(msg); // 'on' o 'off'
  }
});
7

Convención de Topics

Sigue una convención clara para organizar tus topics.

Estructura recomendada:

  • proyecto/dispositivo/estado - Para estados (ej: hogar/living/luz/state)
  • proyecto/dispositivo/set - Para comandos (ej: hogar/living/luz/set)
  • proyecto/sensor/tipo - Para sensores (ej: hogar/sensor/humo)

Mejores Prácticas

  • Usa QoS 1 para mensajes importantes que no pueden perderse
  • Maneja errores siempre con callbacks y eventos 'error'
  • Implementa reconexión automática para conexiones inestables
  • Organiza topics de forma jerárquica y consistente
  • Valida mensajes antes de procesarlos
